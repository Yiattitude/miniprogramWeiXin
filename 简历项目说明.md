# 银发人才平台 - 志愿活动统计小程序

## 1. 项目概述

### 项目背景
该项目是为银发人才（老年志愿者群体）打造的微信小程序平台，旨在帮助老年志愿者便捷地进行志愿活动报名、打卡记录和数据统计，提升志愿服务的数字化管理水平。

### 核心功能模块
- **志愿活动管理**：活动发布、浏览、搜索和筛选
- **报名系统**：在线报名、报名人数实时统计、报名状态管理
- **打卡填报**：服务时长记录、服务人数统计、现场照片上传
- **个人中心**：个人资料管理、志愿服务历史记录查看
- **数据统计**：个人/团队多维度统计分析、数据导出

### 用户群体与特点
- **主要用户**：50岁以上老年志愿者
- **设计特点**：适老化UI设计（大字体、高对比度、简化操作流程）

### 项目规模
- **代码量**：前端约 5000+ 行核心代码
- **页面数**：9个主要功能页面 + 首页
- **组件数**：7个公共组件 + 多个业务组件
- **开发周期**：约 4 周完成核心功能开发与联调

## 3.1 项目负责内容详解

### 3.1.1 架构设计与技术选型
- **技术栈选型**：确定 uni-app + Vue3 + TypeScript 技术方案，确保代码质量和跨平台能力
- **项目架构设计**：建立清晰的目录结构，分离组件、页面、API、工具、状态管理等模块
- **代码规范制定**：统一编码风格、命名规范、注释规范，提升团队协作效率

### 3.1.2 核心功能开发

#### （1）志愿活动模块
- **活动列表页**（`signup-list.vue`）
  - 实现活动列表展示，支持搜索、时间和地点筛选
  - 接入防抖搜索（300ms），优化用户体验
  - 实现上拉加载更多（分页加载）
  
- **活动详情页**（`signup-detail.vue`）
  - 活动详情展示，包含报名人数进度条
  - 报名状态实时同步，已满/已报名状态按钮联动
  - 报名功能实现与状态管理

- **活动发布页**（`publish.vue`，管理员专用）
  - 完整的活动发布表单，包含时间选择、地点输入、人数限制等
  - 使用 uv-form 进行表单校验
  - 管理员权限控制

#### （2）打卡记录模块
- **打卡列表页**（`checkin-list.vue`）
  - 展示用户已报名的活动列表
  - 区分已打卡（置灰）和未打卡状态
  - 点击跳转至打卡填报页

- **打卡填报页**（`checkin-form.vue`）
  - 服务时长、服务人数输入
  - 现场照片上传（最多9张，带尺寸校验）
  - 备注说明填写
  - 完整的表单校验与提交

#### （3）个人中心与统计模块
- **个人主页**（`profile.vue`）
  - 用户信息展示（头像、单位）
  - 三格汇总统计（总时长、总次数、总服务人数）
  - 功能入口菜单

- **历史记录页**（`record.vue`）
  - 历史打卡记录列表，支持分页加载
  - 展示时长、人数、状态徽标

- **统计报表页**（`statistics.vue`）
  - 时间维度 Tab 切换（本月/本年）
  - 个人/团队 Segment 切换
  - 两张统计表格展示
  - 报表导出功能（Excel）

#### （4）首页与导航
- **应用首页**（`index/index.vue`）
  - 功能入口卡片设计
  - 志愿活动模块入口

- **志愿活动首页**（`volunteer/index.vue`）
  - 四格功能入口（打卡/发布/记录/报名）
  - 管理员权限判断，动态显示"发布"入口

## 5. 技术难点与解决方案

### 难点1：适老化UI与标准UI组件的适配
**问题**：uv-ui 默认样式不符合适老化要求
**解决方案**：
- 建立全局 SCSS 变量系统，统一管理颜色、字号、间距
- 封装适配层组件（如 `uv-fallback.vue`），在保持 API 一致的情况下增强样式
- 使用 CSS 深度选择器（`:deep()`）覆盖第三方组件样式

### 难点2：图片上传与压缩
**问题**：老年用户容易上传大尺寸原图，导致上传慢、流量消耗大
**解决方案**：
- 封装 `UploadImage` 组件，集成图片压缩逻辑
- 上传前检测图片尺寸，超过阈值自动压缩（质量80%）
- 显示上传进度，提升用户体验

### 难点3：复杂的权限控制
**问题**：普通用户和管理员看到的页面和功能不同
**解决方案**：
- 在 Pinia User Store 中维护用户角色信息
- 封装 `permission.ts` 工具函数，统一权限判断逻辑
- 在页面中使用 `v-if` 指令控制功能入口显示

### 难点4：统计报表的多维度数据展示
**问题**：需要支持时间维度、身份维度的切换，数据量大
**解决方案**：
- 使用 Tab + Segment 组件实现多维度切换
- 后端返回聚合数据，前端仅负责展示
- 数据缓存策略：同一维度数据缓存 5 分钟，减少重复请求
